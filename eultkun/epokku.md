# エポック

### Introduction

EUL配布プログラムは、2つのフェーズに分かれています。エポック0では、初期のプロトコル利用者に遡ってガバナンストークンが配布され、エポック1では、プロトコル上の借り手に対して継続的にガバナンストークンが配布されます。

### Epoch 0

Epoch 0は、Euler protocolがソフトローンチしていた2021/11/26から2022/03/21までの3ヶ月間である。ユーザーはリスクを最小限に抑えた方法でプロトコルを実験していた。

#### Implementation

この期間にプロトコルを利用した貸し手と借り手には、EULの供給総量の1％のシェアが遡及的に配分された。

配布は以下のように行われました。

1. ブロック14,430,000でプロトコル上の全ユーザーのスナップショットが撮影されました。
2. その時点までにプロトコルと対話した人には、合計271,828EUL（総供給量の1％）が配布されました。
3. アドレスあたりの分配は以下のように計算されます。
   * 発行額の2/3は、当該期間中の預け入れと借り入れの時間加重平均\*に基づいて比例配分されたものです。
   * 発行総額の1/3は、前述の時間帯にプロトコルと相互作用したすべてのユニークなアドレスに均等に分配されます。

_\*簡略化のため、預金および借入金にかかる未収利息は無視した。_

結論として、3407のユニークなアドレスが少なくとも26.59のEULトークンを受け取り、さらに彼らの預金と借入の時間加重平均USD価値に比例した個別の金額を受け取りました。最終的な割り当てについては、[こちら](https://docs.google.com/spreadsheets/d/1jTECkKiMDMvj1UdaU0AEFfhW8IN0SiBfkNFYpTdWIfo/edit#gid=0)をご覧ください。

### Epochs 1-96

エポック1～96は、2022年3月21日から2026年3月11日までの4年間で、この間にどんどん分散化を進めていくことになる。

#### Implementation

この期間にプロトコルを利用した借入人には、EULの総供給量の25%がローリング・マークル分布によって配分される。各エポックでの分配量は、非線形スケジュールで行われる。（下記参照）

各マーケットにおいて、借り手はそのマーケットでの時間加重借入に比例したEULの分配を受けます。各マーケットに割り当てられるEULの量は、EULトークンの保有者によって、エポックごとに決定されます（[Gauges](https://app.gitbook.com/o/-MJloiaY-UMc3SjaxzA6/s/-MJlqpE4apPrZurt7BNr/\~/changes/8TZLu5aIjb41difegSzr/governance/gauges)参照）。 ユーザーは、エポック終了後に、[https://app.euler.finance/](https://app.euler.finance/)の「Claim」ボタンを使って、EUL ガバナンス トークンを請求することができるようになります。

### Schedule

各エポックにおけるEULの配布量は、非線形スケジュールに従って事前に決定されています（以下のグラフを参照）。このスケジュールは、サービス開始後のプロトコル利用者数の増加に合わせてEULの配布量を増加させるもので、初期利用者に不均衡な量を割り当てるのではなく、トークンの供給を分散化するのに役立つはずです。
